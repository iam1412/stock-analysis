## เกณฑ์คัดกรอง (ใช้ตอนวิเคราะห์แต่ละตัว)

| เกณฑ์ | ตัวชี้วัดจริง |
|---|---|
| ไม่เจ๊ง (สำคัญ) | D/E ไม่สูงลิ่ว · OCF เป็นบวกสม่ำเสมอ · ไม่มีธงแดงบัญชี/คดี · มีส่วนแบ่งตลาด/แฟรนไชส์ |
| มาร์จิ้นโตต่อเนื่อง | รายได้&กำไรโต 3–5 ปีเป็นบวก · Gross/Net margin ทรงหรือขยาย (ไม่ผันผวนแบบวัฏจักร) |
| VI ระยะยาว | ROE ≥ ~12% · มี moat · มี Margin of Safety เทียบ Fair Value |
| สภาพคล่อง | A = สูง · B = ใช้ได้ · C = บางลงแต่ยังไม่ตาย |
| ปันผล | โบนัส ไม่บังคับ (หุ้นโตแรงจ่ายน้อยได้) |

**ตัดทิ้ง (ตัวอย่าง):** วัฏจักร PSL/TTA/RCL/KSL/KTIS/BRR/STA/STGT/NER/PTTGC/TPIPL/SPRC/SAT/AH · เสี่ยงเจ๊ง EA/ITD/KEX/SINGER/JMART/SABUY · เพิกถอน INTUCH

## วิธีรัน (ตาม CLAUDE.md §3–4)

- **โมเดล:** All-Sonnet (main + worker) → **escalate เป็น Opus subagent อัตโนมัติ** เฉพาะตัวยาก: IPO <1 ปี (OKJ, COCOCO, KLINIQ, GABLE, AMATAV), spinoff/ควบรวม (SCGD, SJWD, TIPH), หรือราคา/EPS cross-source ขัดกัน
- **1 หุ้น/agent · ≤3 agent/เวฟ · push ทีละเวฟ** (verify → commit → pull --rebase → push HEAD:main)
- **ทุกตัวต้อง:** เริ่มจาก `_template/skeleton-th.html` (฿/SET) · ตรวจข้ามแหล่งราคา+EPS ≥2 · กราฟราคาจริง ~1 ปีรายเดือน (Yahoo `?range=1y&interval=1mo`) · สีแบรนด์เฉพาะตัว · `stock-meta` currency `"THB"` · คำโปรยธุรกิจ `.sub` · meta ai-model · ผ่าน `npm run verify`
- **ลำดับ:** ไล่ **Tier A → B → C** (คุณค่ามาก่อน) · แต่ละเวฟ = 3 ตัวถัดไปตามลำดับ #
- **ก่อนเริ่มแต่ละตัว:** `git pull --rebase origin main` + เช็ค `reports.json` (ถ้ามีสด ≤7 วัน = ข้าม)
- **จังหวะแนะนำ:** ~15–30 ตัว/session → รวม ~5–9 sessions

**เครื่องหมายสถานะ:** ☐ = ยังไม่ทำ · 🔄 = กำลังทำ · ✅ = เสร็จ+push แล้ว (ใส่วันที่)
